# 👨‍💻 Development Guide

This guide explains how to develop neobit-builder locally.

## Project Structure

```
neobit-builder/
├── src/               # Library source (components, screens, hooks, etc.)
├── example/           # Example React Native app
├── cli/               # pop-builder CLI commands
├── lib/               # Built output (generated by bob)
├── package.json       # Main package
└── tsconfig.json
```

## Setup

### 1. Install Dependencies

```bash
# Install neobit-builder deps
npm install

# Install example app deps
cd example
npm install
cd ..
```

### 2. Development Workflow

#### Option A: Using CLI Commands

```bash
# Start example app in dev mode
pop-builder start

# Or specific platform
pop-builder start:ios
pop-builder start:android
```

#### Option B: Direct npm Scripts

```bash
# Example app in dev
npm run example:start

# iOS
npm run example:ios

# Android
npm run example:android
```

#### Option C: Manual Metro Start

```bash
cd example
npm start          # or npm run start:reset
# In another terminal:
npm run ios
npm run android
```

### 3. Building

```bash
# Build library (generates lib/)
npm run build

# Type check (no build)
npm run typecheck

# Clean and rebuild
npm run clean
npm install
npm run build
```

## Key Directories

### `/src/` - Library Source

Main source code organized by domain:

```
src/
├── components/      # UI components (11 total)
├── screens/         # Pre-built auth & app screens
├── navigation/      # Navigation setup
├── theme/           # Design tokens & theme system
├── hooks/           # Custom React hooks
├── store/           # Zustand stores
├── api/             # HTTP client & services
├── services/        # Firebase & auth services
├── i18n/            # Translations
├── types/           # TypeScript types
├── utils/           # Utilities
├── config/          # Config system
├── data/            # Sample/mock data
├── NeobitApp.tsx    # Main component
└── index.ts         # Public API
```

### `/example/` - Example React Native App

A complete, working React Native app demonstrating neobit-builder:

```
example/
├── app.js           # App.tsx alternative (JSX)
├── index.js         # Entry point
├── app.json         # React Native config
├── metro.config.js  # Metro bundler config
├── android/         # Android native project
├── ios/             # iOS native project
└── package.json
```

The example app imports neobit-builder from the parent `src/` directory via Metro's `extraNodeModules` resolver.

## Making Changes

### Adding a New Component

1. Create `src/components/MyComponent/index.tsx`
2. Export from `src/components/index.ts`
3. Export from `src/index.ts`

Example:

```typescript
// src/components/MyComponent/index.tsx
import React from 'react';
import { View, Text } from 'react-native';
import { useAppTheme } from '../../hooks/useAppTheme';

export const MyComponent: React.FC<{ label: string }> = ({ label }) => {
  const { colors } = useAppTheme();
  return (
    <View style={{ backgroundColor: colors.primary }}>
      <Text>{label}</Text>
    </View>
  );
};

export default MyComponent;
```

### Adding a New Screen

1. Create `src/screens/MyScreen/index.tsx`
2. Follow the pattern of existing screens
3. Export from `src/index.ts`

### Modifying Theme

Edit `src/theme/` files:
- `colors.ts` - color definitions
- `light.ts` - light theme overrides
- `dark.ts` - dark theme overrides
- `fonts.ts` - typography
- `spacing.ts` - spacing & sizes
- `index.ts` - exports `getAppTheme()`

### Adding Translations

1. Edit `src/i18n/en.json` for English
2. Edit `src/i18n/ar.json` for Arabic
3. Use in components: `const { t } = useTranslation()`

## Testing

### Type Checking

```bash
npm run typecheck
```

### Run on Device

```bash
# iOS simulator
npm run example:ios

# Android emulator
npm run example:android

# Real device
npm run example:ios -- --device    # iOS
npm run example:android            # Android (auto-detects device)
```

### Testing in Example App

The example app uses Metro's module resolution to directly import from `src/`:

```javascript
// example/metro.config.js
resolver: {
  extraNodeModules: {
    'neobit-builder': '/path/to/neobit-builder',
  },
}
```

Changes in `src/` are hot-reloaded automatically. No need to rebuild!

## Tips & Tricks

### Hot Reload

Changes in `src/` are automatically reloaded in the example app when Metro is running. No rebuild needed!

### Debugging Screens

The example app shows all screens in a single interface. To test a specific screen, modify `example/app.js`:

```javascript
import { LoginScreen } from 'neobit-builder';

export default function App() {
  return <LoginScreen navigation={{}} route={{}} />;
}
```

### Running on Device

```bash
# Get your IP
ifconfig | grep "inet " | grep -v 127.0.0.1

# Update metro.config.js
watchFolders: ['...', '--max-workers=4'],

# Connect device to same WiFi, then:
npm run example:ios -- --device
```

### Environment Variables

For API base URL switching:

```typescript
// src/api/config.ts
const API_ENV: ApiEnv = process.env.API_ENV || 'production';

// .env (example)
API_ENV=local
LOCAL_IP=192.168.1.5
```

## Troubleshooting

### "Cannot find module 'neobit-builder'"

Make sure `example/metro.config.js` has the correct `extraNodeModules` entry pointing to parent directory.

### TypeScript errors in screens

Run `npm run typecheck` to see all errors at once.

### Build fails with "bob"

```bash
# Reinstall build tools
npm install --save-dev react-native-builder-bob
npm run build
```

### iOS pods won't install

```bash
cd example/ios
rm -rf Pods Podfile.lock
pod install
cd ../..
npm run example:ios
```
